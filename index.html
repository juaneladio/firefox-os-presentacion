¿Qué navegadores usamos hoy?

Laboratorio de informática en Lima, Perú
Es una pregunta general.

Mozilla hizo posible elegir

En el origen de Mozilla existió un estancamiento de las tecnologías web. Mozilla cambió el panorama y permitió que hoy podamos elegir entre muchos navegadores de Internet, no quedarnos con uno de fábrica. Firefox (y Opera) hicieron posible que los navegadores no sólo mejoren sino también que soporten estándares web comunes entre todos.

Pero fue sólo el comienzo …

Mozilla es más que Firefox. Nuestra visión es que Internet sea abierto y accesible para todos, y eso es más que fabricar un navegador. Esto nos ha llevado a educar a la gente sobre cómo se publica contenido en la web y protegemos la privacidad y la posibilidad de elección en Internet.

Mozilla Webmaker

webmaker.org

Mozilla Webmaker es un proyecto en marcha para volver a los ciudadanos de a pie, que no estudiaron Computación, sobre cómo se puede crear una web, cómo se pueden publicar páginas, remezclar video y métodos simples para mantenerse seguro mientras navegan.

Un nuevo frente

Photo by Jason Weaver
Desde hace mucho la tendencia es el mundo móvil, y la posibilidad de acceder a Internet desde un celular, una tablet, cualquier dispositivo móvil. No sólo aparecen nuevos equipos cada mes, sino que también tienen diferentes tamaños y resoluciones de pantalla. Esto nos lleva a pensar en el reto de movernos hacia una web móvil.

HTML5 para el mundo móvil

En el principio la promesa era que HTML5 permitiría a los desarrolladores crear aplicaciones web que le saquen provecho a los recursos de un celular. Finalmente las aplicaciones web tendrían una plataforma que no sólo estaría a la vanguardia sino también les permitiría a todos construir para estos dispositivos.

No, aplicaciones nativas

Photo by Michael Gil
Pero la promesa no se cumplió porque las empresas notaron que sería más fácil lucrar en entornos cerrados en tiendas de aplicaciones, con programas 'nativos' que sólo pudiesen ejecutarse en una plataforma propia. Si se hubiese permitido usar tecnologías abiertas, perderían el control y tendrían menos márgenes de ganancia. Como desarrolladores no deberíamos seguir ese camino.

La primera respuesta

La primera respuesta de Mozilla en el frente móvil fue Firefox para Android, un navegador que llevara parte de las ventajas de Firefox de escritorio a un móvil, y que permitiera sincronizar datos de navegación entre móviles y PCs. Está disponible en Google Play y tiene más opciones que el Android Browser que viene por defecto. No hay Firefox para otras plataformas móviles por impedimentos legales.

Herramientas para desarrolladores en Firefox

Photo by Clive Darra
Otra iniciativa son las Herramientas para desarrolladores en Firefox, que proveen muchas funcionalidades útiles para desarrolladores de webs móviles y webs en general, como el Inspector, la consola web, entre otros.

Vista de diseño adaptable

En las Herramientas para desarrolladores en Firefox se dispone de una opción móvil: la vista de diseño adaptable, que permite simular la vista de páginas en dispositivos móviles, la rotación del dispositivo, y todo sin perder los botones normales de un Firefox para escritorio.

Depuración remota PC > móvil

hacks.mozilla.org

Otra herramienta es la depuración remota desde un Firefox de escritorio hacia un Firefox para Android, conectado vía cable USB. Así, puedes hacer los cambios en desktop y ver cómo va cambiando la aplicación web móvil sin tener que recargar la página o reiniciar el equipo.

Pero es un callejón sin salida

Photo by Doug Geisler
Pero todas estas iniciativas no eran suficientes si no hay la posibilidad de acceder desde un navegador al hardware de un móvil: la cámara, el GPS, la batería, etcétera. La posibilidad sólo existe en aplicaciones nativas y no es justo que se hable de las posibilidades futuristas de HTML5 cuando se tienen estas serias limitaciones.

La siguiente respuesta: Web API

arewemobileyet.com

Es por eso que comenzamos a crear la Web API: una interfaz de programación que nos permitiera aprovechar todo el acceso posible a los componentes de un móvil, y que esta interfaz pudiera estandarizarse y ser implementada por más fabricantes de navegadores. En arewemobileyet.com se muestra cómo va la definición e implementación de la interfaz, y así solo sea necesario usar JavaScript para acceder a los componentes de un móvil.

La respuesta final: Firefox OS

Pero seguía siendo insuficiente, dado que dependíamos de la buena voluntad de empresas fabricantes de sistemas operativos móviles. Es así como el reto verdadero es crear un Sistema Operativo que nos permitiese no sólo hacer llamadas y navegar en Internet, sino que le de la posibilidad a empresarios y desarrolladores de crear aplicaciones que puedan ejecutarse en cualquier lugar. Esto es Firefox OS.
Photo by Alesa Dam
Sólo como referencia: Firefox OS está compuesto por una capa de bajo nivel (Gonk), Gecko que es el motor de Firefox de siempre, y equipado con las interfaces para acceder a los componentes del celular y Gaia, una interfaz que puede ser personalizada por fabricantes y operadores de telefonía. Las aplicaciones se programan hacia Gecko usando tecnologías web, no necesitas aprender un nuevo lenguaje.

Interfaz realmente flexible

Gaia es la interfaz de Firefox OS por defecto, pero un fabricante o un operador de telefonía puede personalizar completamente la interfaz. Así, podríamos tener dispositivos con experiencias personalizadas para distintos públicos, pero usando la misma base.

Geeksphone: un teléfono para desarrolladores

geeksphone.com

Este es el primer dispositivo puesto a la venta con Firefox OS por Geeksphone, una empresa española. Son celulares baratos que pueden enviarse a cualquier parte del mundo y que fueron diseñados para ser 'hackeables' por programadores.

No estamos solos

www.mozilla.org/en-US/firefox/partners/

Pero Mozilla quiere que esto sea una iniciativa real, y comenzó a contactar a empresas fabricantes de celulares y proveedores de telefonía, para que comiencen a ver Firefox OS como una alternativa real. Ahora son decenas de empresas que apuestan en Firefox OS.

Foxconn

itworld.com

La última noticia llega de Foxconn, empresa taiwanesa especializada en fabricación de móviles, que ahora no sólo desarrolla cinco dispositivos con Firefox OS, sino que también va a contratar a 3000 personas para desarrollar alrededor de Firefox OS.

Telefónica

blogthinkbig.com

Un aliado importante es Telefonica, un operador de telefonía español que se involucró al conocer la existencia del proyecto y que ha colaborado no sólo en el desarrollo del código de Firefox OS, sino que será la primera empresa que venderá dispositivos a consumidores en sus mercados de Latinoamérica y Europa.

Administrador de aplicaciones de Firefox / Firefox OS Simulator

La forma más sencilla de probar Firefox OS es instalando Firefox y luego Firefox OS Simulator, un complemento que simula un equipo, y en el cual se pueden instalar aplicaciones. Puedes 'hackear' otros celulares pero el proceso es complicado. La idea de Firefox OS es traer nuevos celulares con acceso a la web, no reemplazar a Android/iOS.

Aplicaciones web en teléfonos

marketplace.firefox.com

Firefox OS tiene un primer Firefox Marketplace, un lugar donde se tengan aplicaciones gratuitas y de pago, y que ya tiene algunas aplicaciones básicas. Éstas se pueden probar no sólo en Firefox OS sino también en Firefox Para Android y Firefox para escritorio. Las aplicaciones se construyen con HTML5, CSS y JavaScript, y sólo necesitan un archivo de texto extra: un manifest.

developer.mozilla.org

Paso 5: Publica

marketplace.firefox.com
